# E05 - Quality and Release

## Objetivo
Garantizar calidad mínima antes de release del MVP.

## Estrategia de pruebas
- Unit tests Dart por capa.
- Unit tests Rust de exactitud y robustez.
- Integración Flutter-Rust en casos de entrada/salida controlados.
- Validación Web para implementación Dart de engine.

## Gates de calidad
- Lint limpio.
- Tests verdes en CI.
- Checklist de release completado.

## Release checklist
- Permisos y mensajes de error validados.
- Flujo start/stop estable.
- Calibración A4 validada.
- Casos de audio ruidoso evaluados.

## Criterios de aceptación
- Condición Go/No-Go documentada.
- Evidencias mínimas enlazadas desde tasks.
